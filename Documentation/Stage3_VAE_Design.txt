1. Stage 3 Overview - Role in the Pipeline

Stage 3 is the architectural bridge between classical preprocessing and quantum computation. Its sole responsibility is to compress the 167-feature (or ~155 after variance filtering) normalised feature vectors produced by Stage 2 into 8-dimensional latent representations, where each dimension is bounded to the interval [0, p]. These 8 values become the rotation angles applied to 8 qubits in Stage 4's Variational Quantum Circuit. Every design decision in Stage 3 must therefore be evaluated against one ultimate criterion: does this produce 8 meaningful, well-distributed, angle-compatible values that maximise the VQC's classification capacity?

The model chosen for this compression task is a Variational Autoencoder (VAE). A VAE is selected over alternatives (PCA, standard autoencoder, UMAP) for three reasons that are directly relevant to the paper's contributions: (1) it produces a probabilistic latent space, not a deterministic mapping, giving the quantum layer richer angle diversity; (2) its learned encoder is invertible in principle, supporting the explainability claims of Stage 6; and (3) it produces a structured, regularised latent space via the KL divergence term - preventing the collapsing or clustering of latent representations that would starve the VQC of discriminative signal.

2.1  Loading Strategy - Memory-Safe Data Pipeline
The training set (2,367,588 Ś 167 float32) occupies 1.58 GB in RAM when fully loaded. The sentinel mask (same shape, bool) adds another 395 MB. Loading both simultaneously consumes 1.97 GB before any model weights or gradients are allocated. On a system with 8 GB RAM and a GPU with 8 GB VRAM, naive full-load will cause out-of-memory errors.

The solution is a custom PyTorch Dataset class that reads from Parquet row groups on demand and streams mini-batches to the DataLoader. Parquet files are stored in row groups of ~100,000 rows by default (matching the chunk size used in Stage 2's save step), making row-group streaming efficient.

    Loads one row-group at a time to keep peak RAM under control.
    All data returned as float32 tensors.

        self.mask_path  = mask_path
        self.label_path = label_path
        # var_filter_cols: list of column names to DROP for VAE-B
        self.var_filter = var_filter_cols or []
 
        # Load fully into RAM only if < 2 GB - otherwise use chunked __getitem__
        # For 2.37M rows this is borderline; use chunked strategy always.
        self._load_to_ram()
 
    def _load_to_ram(self):
        import pyarrow.parquet as pq
        print(f'Loading {os.path.basename(self.x_path)} ...')
        X_df   = pq.read_table(self.x_path).to_pandas()
        mask_df = pq.read_table(self.mask_path).to_pandas()
 
        if self.var_filter:
            drop = [c for c in self.var_filter if c in X_df.columns]
            X_df    = X_df.drop(columns=drop)
            mask_df = mask_df.drop(columns=drop)
            print(f'  VAE-B: dropped {len(drop)} near-zero variance columns')
 
        self.X    = torch.tensor(X_df.values,    dtype=torch.float32)
        self.mask = torch.tensor(mask_df.values, dtype=torch.bool)
        self.n_features = self.X.shape[1]
 
        if self.label_path:
            y_df = pd.read_parquet(self.label_path)
            self.y = torch.tensor(y_df['y'].values, dtype=torch.long)
        else:
            self.y = None
 
        print(f'  X shape: {self.X.shape}  mask shape: {self.mask.shape}')
 
    def __len__(self):
        return len(self.X)
 
    def __getitem__(self, idx):
        if self.y is not None:
            return self.X[idx], self.mask[idx], self.y[idx]
        return self.X[idx], self.mask[idx]

3. VAE Architecture - Complete Specification

The VAE is a generative model that learns to compress high-dimensional input into a low-dimensional probabilistic latent space. Unlike a standard deterministic autoencoder, the VAE encoder outputs the parameters of a Gaussian distribution (mean ” and log-variance log sČ) over the latent space. This probabilistic structure provides two critical properties for the quantum pipeline: (1) the latent space is continuous and structured, preventing isolated point clusters that would produce degenerate quantum angle distributions; and (2) the reparameterisation trick allows gradient flow through the sampling step, making end-to-end training stable.

The decoder ALWAYS receives z (not z_angle). z_angle is derived from ” post-training for downstream use only.
3.2  Why This Specific Architecture - All Design Decisions Justified
3.2.1  Layer Widths: INPUT_DIM ? 256 ? 128 ? 64 ? 8
The encoder follows a progressive halving pattern (256 ? 128 ? 64) before the bottleneck. This is deliberate: abrupt compression (e.g. INPUT_DIM ? 8 in one step) forces the network to discard information before it has extracted meaningful abstractions. Each intermediate layer can be thought of as a progressively coarser feature map - the 256-dim layer learns pairwise feature interactions; the 128-dim layer learns higher-order patterns; the 64-dim layer distils these into a near-bottleneck representation; the 8-dim bottleneck selects the 8 most informative signal dimensions.

The decoder mirrors the encoder (8 ? 64 ? 128 ? 256 ? INPUT_DIM) for symmetry. A symmetric decoder is not strictly required but gives the reconstruction path the same representational power as the encoding path, which produces more informative reconstruction loss gradients.

3.2.2  Why GELU Activation over ReLU
ActivationBehaviourDecisionReLUf(x) = max(0, x). Hard zero for all negative inputs.REJECTED. Dying ReLU problem: with sentinel-rich inputs (many features = 0.0), neurons receiving mostly-zero activations can permanently die, losing gradient flow through entire latent dimensions.Sigmoid / TanhSaturates at extremes. Gradient vanishes.REJECTED for hidden layers. Saturation causes vanishing gradients in the deeper encoder layers. Only appropriate for the final decoder output (bounding to [0,1]).GELUf(x) = x · F(x) where F is the Gaussian CDF. Smooth, non-monotone near zero.CHOSEN. GELU is the standard activation in Transformer and BERT architectures precisely because it does not fully zero negative activations - it applies a smooth gate, preserving gradient flow even for slightly-negative inputs. Especially important here where many inputs are 0.0 (sentinel) or near-0 (normalised low-value features).
3.2.3  Batch Normalisation - Where to Use and Where Not To
Use BatchNorm: Between encoder hidden layers (after each linear layer, before activation). BatchNorm accelerates training by reducing internal covariate shift and acts as a mild regulariser.
Do NOT use BatchNorm on the ” and log sČ outputs. BatchNorm would normalise the mean outputs, shifting their distribution. Since sigmoid(”) Ś p maps ” to qubit angles, a normalised ” would produce angle distributions centred around p/2 regardless of the actual data. This would severely limit the VQC's ability to discriminate classes that should produce angles near 0 or near p.
Do NOT use BatchNorm on the bottleneck z. Same reason - z is derived from ”, and any post-hoc normalisation of z changes the angle distribution unpredictably.
Do NOT use BatchNorm on the decoder output layer. The decoder output must be in [0, 1] (via sigmoid). BatchNorm before sigmoid would shift the distribution away from [0, 1].

3.2.4  Dropout - Optional Regularisation
Dropout of p=0.1-0.2 can be applied in the encoder hidden layers only. It should NOT be applied after the ”/log sČ heads or in the decoder. Reason: the VAE already has a regularisation mechanism (the KL divergence term). Adding heavy dropout can over-regularise the latent space, making it difficult for the VAE to produce sufficiently varied angle distributions for the VQC.

3.3  The Quantum-Safe Bottleneck - Detailed Specification
This is the single most important design element of Stage 3. The VQC in Stage 4 uses angle encoding: each latent dimension ?? is used as the rotation angle of the R? gate applied to qubit i. The RY gate rotation is physically bounded to [0, 2p] for a full rotation, but the parameterised range [0, p] is chosen because it spans the full Bloch sphere hemisphere without wrapping - avoiding degenerate cases where ?  0 and ?  2p produce identical qubit states.

Bounding MethodFormulaWhy Chosen / RejectedHard cliptorch.clamp(”, 0, p)REJECTED. Clipping kills gradients at the boundary - any ” > p or ” < 0 produces zero gradient and no learning signal. def reparameterise(mu, log_var):
    """Standard reparameterisation trick for VAE training."""
    std = torch.exp(0.5 * log_var)
    eps = torch.randn_like(std)   # eps ~ N(0, I)
    return mu + eps * std         # z ~ N(mu, sigma^2)
 
def to_quantum_angles(mu):
    """
    Convert encoder mean output to qubit rotation angles.
    Output guaranteed in (0, pi) - open interval, never exactly 0 or pi.
    Use this ONLY at inference time (after training), not during training.
    """
    return torch.sigmoid(mu) * torch.pi

3.4  Full VAE Class - Complete Implementation
class SentinelAwareVAE(nn.Module):
    """
    VAE with sentinel-aware reconstruction loss.
    Supports both standard VAE (cond_dim=0) and
    conditional VAE (cond_dim=5 for 5-class one-hot label).
    """
    def __init__(self, input_dim, latent_dim=8, hidden_dims=None, cond_dim=0):
        super().__init__()
        self.input_dim  = input_dim
        self.latent_dim = latent_dim
        self.cond_dim   = cond_dim
        enc_in = input_dim + cond_dim   # cVAE: concat label to input
 
        if hidden_dims is None:
            hidden_dims = [256, 128, 64]
 
        # -- Encoder ---------------------------------------------
        enc_layers = []
        in_d = enc_in
        for h in hidden_dims:
            enc_layers += [
                nn.Linear(in_d, h),
                nn.BatchNorm1d(h),   # BatchNorm between layers - OK
                nn.GELU(),
                nn.Dropout(0.1),
            ]
            in_d = h
        self.encoder = nn.Sequential(*enc_layers)
        self.fc_mu      = nn.Linear(in_d, latent_dim)  # no BN, no activation
        self.fc_log_var = nn.Linear(in_d, latent_dim)  # no BN, no activation
 
        # -- Decoder ---------------------------------------------
        dec_in = latent_dim + cond_dim  # cVAE: also concat label to decoder
        dec_layers = []
        rev_dims = list(reversed(hidden_dims))
        in_d = dec_in
        for h in rev_dims:
            dec_layers += [
                nn.Linear(in_d, h),
                nn.BatchNorm1d(h),
                nn.GELU(),
            ]
            in_d = h
        dec_layers.append(nn.Linear(in_d, input_dim))
        dec_layers.append(nn.Sigmoid())   # output in [0, 1]
        self.decoder = nn.Sequential(*dec_layers)
 
    def encode(self, x, c=None):
        if c is not None:
            x = torch.cat([x, c], dim=1)
        h   = self.encoder(x)
        mu  = self.fc_mu(h)
        lv  = self.fc_log_var(h)
        return mu, lv
 
    def decode(self, z, c=None):
        if c is not None:
            z = torch.cat([z, c], dim=1)
        return self.decoder(z)
 
    def forward(self, x, mask, c=None):
        mu, log_var = self.encode(x, c)
        z           = reparameterise(mu, log_var)  # used for decoding
        x_recon     = self.decode(z, c)
        return x_recon, mu, log_var
 
    def get_quantum_angles(self, x, c=None):
        """Inference only - returns deterministic angle vector."""
        self.eval()
        with torch.no_grad():
            mu, _ = self.encode(x, c)
            return to_quantum_angles(mu)   # shape (batch, 8), values in [0, pi]

4. Loss Function - Sentinel-Masked ELBO

The VAE is trained by maximising the Evidence Lower BOund (ELBO), which is equivalent to minimising the negative ELBO. The ELBO decomposes into two terms: a reconstruction term and a KL divergence regularisation term. Both terms require special treatment given the sentinel structure of the data.

4.1  Reconstruction Loss - Why Masking Is Non-Negotiable
The reconstruction loss measures how well the decoder recovers the input from the latent code. In standard MSE reconstruction loss applied to the full input vector, sentinel positions (value = 0.0) contribute equally to the loss as real features. Since NSL-KDD records have 76.6% sentinel rate and UNSW-NB15 has 72.7%, the majority of reconstruction loss for those records is generated by sentinel positions - which are trivially easy to reconstruct (always output 0.0) and carry no information.

If sentinel positions dominate the reconstruction loss, the encoder learns to compress information that explains 'which dataset did this record come from' rather than 'what attack pattern does this record exhibit'. The bottleneck dimensions then encode dataset identity, not threat semantics. The quantum angles would separate the three datasets, not the five threat classes.

def vae_loss(x, x_recon, mu, log_var, sentinel_mask, beta=1.0):
    """
    Sentinel-masked ELBO loss.
 
    Parameters
    ----------
    x            : (batch, F) float32 - original input
    x_recon      : (batch, F) float32 - decoder output
    mu           : (batch, 8) float32 - encoder mean
    log_var      : (batch, 8) float32 - encoder log-variance
    sentinel_mask: (batch, F) bool    - True = absent feature
    beta         : float - KL weight (beta-VAE if > 1.0)
 
    Returns
    -------
    total_loss, recon_loss, kl_loss  (all scalars)
    """
    # Weight: 1.0 for real features, 0.0 for sentinel positions
    weight = (~sentinel_mask).float()              # (batch, F)
 
    # Per-element squared error, zeroed at sentinel positions
    sq_err = weight * (x - x_recon).pow(2)        # (batch, F)
 
    # Sum over features, mean over batch
    # Normalise by number of REAL features per sample to avoid
    # penalising NSL-KDD records (76.6% sentinel) more than CICIDS records
    real_count = weight.sum(dim=1).clamp(min=1)   # (batch,) - avoid /0
    recon_loss = (sq_err.sum(dim=1) / real_count).mean()
 
    # KL divergence: D_KL(N(mu, sigma^2) || N(0,1))
    # Analytic form - no masking needed
    kl_loss = -0.5 * (1 + log_var
                         - mu.pow(2)
                         - log_var.exp())
    kl_loss = kl_loss.sum(dim=1).mean()   # sum over latent dims, mean over batch
 
    total_loss = recon_loss + beta * kl_loss
    return total_loss, recon_loss, kl_loss
Normalisation by Real Feature Count
The recon_loss formula divides the per-sample squared error by real_count (number of non-sentinel features for that sample).
Without this normalisation: NSL-KDD records (76.6% sentinel = ~41 real features) contribute ~41 terms to the loss, while CICIDS-2017 records (50.0% sentinel = ~84 real features) contribute ~84 terms. The gradient is dominated by CICIDS-2017.
With normalisation: each record contributes equally to the mean loss regardless of how many of its features are real. The VAE trains uniformly across all three datasets.
This is the correct statistical approach and directly supports the paper's claim that the model generalises across heterogeneous data sources.
4.2  KL Divergence - The ß Parameter
The KL divergence term D_KL(N(”, sČ) || N(0,I)) penalises the encoder for producing latent distributions that deviate from the unit Gaussian prior. This regularisation pressure forces the encoder to produce overlapping, continuous latent distributions - preventing the 'posterior collapse' where each sample gets a unique isolated point in latent space.

ß = 1.0 (standard VAE): Balanced reconstruction and regularisation. Start here.
ß < 1.0 (reconstruction-heavy): Prioritises reconstruction quality over latent structure. Risk: posterior collapse - the KL term is too weak to prevent the encoder from ignoring the prior.
ß > 1.0 (ß-VAE): Stronger regularisation ? more disentangled latent space (each dimension encodes an independent factor). Recommended range ß ? [2, 5] if initial experiments show poor latent separation. Risk: too-high ß causes over-regularisation - the encoder maps all inputs to N(0,I), making the latent code useless.

ß Tuning Strategy for IEEE-Quality Results
Run initial training with ß=1.0. After training, compute the per-class latent space separation metric (mean pairwise class distance in 8D space).
If MALWARE and NORMALL clusters overlap substantially (mean distance < 0.5), increase ß to 2.0 and retrain.
Report the ß value used for both VAE-A and VAE-B in the paper. If different ß values produce optimal results for the two models, this is itself an interesting finding about how feature quality affects the VAE's regularisation needs.
The beta-VAE framework was established by Higgins et al. (2017) - cite this when describing your ß choice.
4.3  KL Annealing - Preventing Posterior Collapse at Training Start
A known failure mode in VAE training is posterior collapse at the start of training. In the first few epochs, the decoder is random and reconstruction loss is very high. The KL term, starting at zero (”=0, log sČ=0), provides almost no gradient. The encoder quickly learns to ignore the prior and produce deterministic ” values to minimise reconstruction loss - then the KL term suddenly spikes and training destabilises.

KL annealing solves this by starting ß at 0.0 and gradually increasing it to the target value over a warmup period:

class KLAnnealer:
    """
    Linearly anneals beta from 0 ? beta_target over warmup_epochs,
    then holds at beta_target.
    """
    def __init__(self, beta_target=1.0, warmup_epochs=10):
        self.beta_target    = beta_target
        self.warmup_epochs  = warmup_epochs
 
    def get_beta(self, epoch):
        if epoch >= self.warmup_epochs:
            return self.beta_target
        return self.beta_target * (epoch / self.warmup_epochs)
 
# Usage in training loop:
annealer = KLAnnealer(beta_target=1.0, warmup_epochs=10)
for epoch in range(n_epochs):
    beta = annealer.get_beta(epoch)

5.2  Training Loop - Full Implementation
def train_vae(model, train_loader, val_loader, config, model_name='VAE-A'):
    """Complete training loop with all stability measures."""
    device    = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model     = model.to(device)
    optimizer = torch.optim.AdamW(model.parameters(),
                                  lr=config['lr'],
                                  weight_decay=config['weight_decay'])
    scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
        optimizer, mode='min', factor=0.5, patience=5, verbose=True)
    annealer  = KLAnnealer(config['beta_target'], config['warmup_epochs'])
 
    best_val_loss = float('inf')
    best_epoch    = 0
    patience_ctr  = 0
    history       = []
 
    for epoch in range(1, config['max_epochs'] + 1):
        beta = annealer.get_beta(epoch - 1)
 
        # -- Training phase -----------------------------------
        model.train()
        tr_total = tr_recon = tr_kl = 0.0
        n_batches = 0
 
        for batch in train_loader:
            x, mask = batch[0].to(device), batch[1].to(device)
            c = None
            if model.cond_dim > 0:   # cVAE: one-hot label
                y = batch[2].to(device)
                c = torch.zeros(len(y), 5, device=device)
                c.scatter_(1, y.unsqueeze(1), 1.0)
 
            x_recon, mu, log_var = model(x, mask, c)
            loss, recon, kl = vae_loss(x, x_recon, mu, log_var, mask, beta)
 
            optimizer.zero_grad()
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()
 
            tr_total += loss.item();  tr_recon += recon.item();  tr_kl += kl.item()
            n_batches += 1
 
        tr_total /= n_batches;  tr_recon /= n_batches;  tr_kl /= n_batches
 
        # -- Validation phase ---------------------------------
        model.eval()
        val_total = val_recon = val_kl = 0.0
        n_val = 0
        with torch.no_grad():
            for batch in val_loader:
                x, mask = batch[0].to(device), batch[1].to(device)
                c = None
                if model.cond_dim > 0:
                    y = batch[2].to(device)
                    c = torch.zeros(len(y), 5, device=device)
                    c.scatter_(1, y.unsqueeze(1), 1.0)
                x_recon, mu, log_var = model(x, mask, c)
                loss, recon, kl = vae_loss(x, x_recon, mu, log_var, mask, beta)
                val_total += loss.item();  val_recon += recon.item();  val_kl += kl.item()
                n_val += 1
        val_total /= n_val;  val_recon /= n_val;  val_kl /= n_val
 
        scheduler.step(val_recon)   # schedule on masked recon loss
 
        # -- Logging -----------------------------------------
        history.append({'epoch':epoch,'beta':beta,
            'tr_total':tr_total,'tr_recon':tr_recon,'tr_kl':tr_kl,
            'val_total':val_total,'val_recon':val_recon,'val_kl':val_kl})
 
        if epoch % 5 == 0 or epoch == 1:
            print(f'[{model_name}] E{epoch:03d} ß={beta:.2f} | '
                  f'tr_recon={tr_recon:.4f} tr_kl={tr_kl:.4f} | '
                  f'val_recon={val_recon:.4f} val_kl={val_kl:.4f}')
 
        # -- Early stopping & checkpoint ----------------------
        if val_recon < best_val_loss - 1e-6:
            best_val_loss = val_recon
            best_epoch    = epoch
            patience_ctr  = 0
            torch.save({'epoch':epoch,'model_state':model.state_dict(),
                        'optimizer_state':optimizer.state_dict(),
                        'val_recon':val_recon,'config':config},
                       f'{STAGE3_DIR}/{model_name}_best.pt')
        else:
            patience_ctr += 1
            if patience_ctr >= config['patience']:
                print(f'[{model_name}] Early stop at epoch {epoch}. Best: epoch {best_epoch} val_recon={best_val_loss:.4f}')
                break
 
    return history

Production Inference Warning for cVAE
In production, when classifying a new network flow, its label is unknown (that's what we're trying to predict). A cVAE encoder requires the label at inference time.
Solutions: (1) Use the unconditional VAE encoder (set c=None) at inference - the model was trained conditionally but can be queried without conditioning. Quality degrades slightly.
(2) Two-pass inference: run a fast classical pre-classifier (logistic regression on raw features) to get a predicted label, then use that prediction as c for the cVAE encoder.
(3) Marginalise: compute encoder output for all 5 possible labels and average the resulting latent vectors. Computationally expensive but theoretically sound.
Document which strategy is used in the paper's system design section.

8. Potential Problems - Complete Catalogue with Solutions

VAE training on multi-source, sentinel-rich, class-imbalanced network traffic data introduces numerous failure modes that do not appear in standard VAE tutorials. This section documents every known problem, its symptom, its root cause, and its solution.

P-01 - Posterior Collapse
Symptom
KL loss drops to near-zero after a few epochs and stays there.
Reconstruction loss plateaus at a high value.
All input samples map to nearly identical latent vectors (”  0, s  1).
The quantum angle vector is nearly identical for all samples: z_angle  [p/2, p/2, p/2, p/2, p/2, p/2, p/2, p/2].
Root cause: The KL term overpowers the reconstruction term early in training. The encoder learns to set ”=0, log sČ=0 for all inputs (perfectly matching the prior) to minimise KL loss, ignoring the reconstruction signal.

Solutions (apply in order)
Solution 1 - KL Annealing: Use the KLAnnealer class (Section 4.3). Start ß=0 and ramp to ß=1.0 over 10 epochs. This is the primary fix.
Solution 2 - Reduce ß: Decrease ß_target to 0.5. Monitor if KL loss stabilises above 0.1 per latent dim.
Solution 3 - Free bits (Kingma et al. 2016): Set a minimum KL budget per latent dim (e.g. max(KL_per_dim, 0.25)). The KL term only penalises dims whose KL exceeds the budget, leaving others free to collapse.
Solution 4 - Increase batch size to 1024. Larger batches provide more stable gradient estimates and reduce the spike in reconstruction loss that triggers the KL-dominance feedback loop.
P-02 - Latent Space Clustering by Dataset, Not by Class
Symptom
t-SNE visualisation of the 8D latent space shows 3 well-separated clusters corresponding to NSL-KDD, UNSW-NB15, and CICIDS-2017, not 5 clusters for threat classes.
VQC classification accuracy is high between datasets but poor between threat classes within the same dataset.
Root cause: The sentinel pattern is the strongest structural signal in the data. NSL-KDD has 76.6% sentinel (118/167 features = 0.0). UNSW-NB15 has 72.7%. CICIDS-2017 has 50.0%. If the sentinel mask is not applied to the reconstruction loss, the VAE's easiest path to low reconstruction loss is to encode which 118/112/77 features are zero - i.e., to encode dataset identity rather than threat semantics.

Solutions
Primary fix: Ensure the sentinel mask is correctly applied in the loss function (Section 4.1). Verify weight = (~sentinel_mask).float() before training.
Diagnostic: Compute the mean latent vector per source dataset (”_NSL, ”_UNSW, ”_CICIDS). If these are well-separated in 8D, the problem is present.
Secondary fix: Use conditional VAE (cVAE) conditioned on master_label. This directly guides the encoder to organise latent space by threat class.
Tertiary fix: Add a classification auxiliary loss (multi-task learning). After the bottleneck, add a 5-class softmax head trained jointly with the VAE loss. This forces the latent dimensions to encode threat-class-relevant information.
P-03 - Quantum Angle Collapse (All Angles  p/2)
Symptom
After training, z_angle = sigmoid(”) Ś p has very low standard deviation across all 8 dimensions.
Per-dimension std < 0.1 for most dimensions.
VQC receives nearly identical angle vectors for different samples, making classification impossible.
Root cause: sigmoid(0) = 0.5, so sigmoid(0) Ś p = p/2. If the encoder produces ”  0 for all inputs (posterior collapse, P-01), all angles collapse to p/2. Even without full collapse, if ” values are small (e.g. in [-0.5, 0.5]), sigmoid compression maps them to a narrow band around p/2.

Solutions
Primary fix: Fix posterior collapse (P-01) first. Angle collapse is a symptom, not the cause.
Monitor per-dimension std of ” during training. Healthy training shows ” values spread across [-3, +3], giving angle std  0.4-0.8 radians per dimension.
If ” values are too small, reduce the weight decay (try 1e-5 instead of 1e-4). Weight decay penalises large weights and can prevent the encoder from producing spread-out ” values.
If specific dimensions collapse: add a per-dimension KL floor (free bits, see P-01 Solution 3). This ensures every dimension is forced to carry some information.
P-04 - Out-of-Memory During Training
Symptom
CUDA out of memory (OOM) error during training.
Or: system RAM exhausted when loading train Parquet file.
Solutions
GPU OOM: Reduce batch size from 512 to 256 or 128. Gradient accumulation can compensate: accumulate gradients over 4 batches of 128 before calling optimizer.step(), giving an effective batch size of 512.
CPU RAM: The SentinelAwareParquetDataset loads both X_train (1.58 GB) and mask_train (395 MB) = 1.97 GB. Use fp16 (half-precision) for X_train storage: X = X.half(). Cast to float32 inside the training loop. This halves RAM to ~990 MB.
If still OOM: implement row-group streaming. Load one Parquet row group (~100K rows) at a time. Each epoch shuffles the row group order.
Check: use torch.cuda.memory_summary() after the first batch to diagnose GPU memory allocation.
P-05 - NaN Loss During Training
Symptom
loss.item() returns nan after a few epochs.
Or: gradients contain nan, causing all parameters to become nan.
Root cause candidates: (1) log_var blows up: exp(log_var) = sČ overflows if log_var > 88 (float32 max). (2) Division by zero in recon_loss normalisation if real_count = 0. (3) Gradient explosion in first epochs before learning rate warmup stabilises.

Solutions
Clamp log_var: add log_var = torch.clamp(log_var, min=-10, max=10) in the encode() method. This bounds sČ to [e?č°, eč°]  [4.5e-5, 22026], preventing overflow.
Guard real_count: ensure real_count = weight.sum(dim=1).clamp(min=1) (already in the provided loss function). Never divide by zero.
Gradient clipping: ensure torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) is called before optimizer.step().
Add torch.autograd.detect_anomaly() context manager in the first 5 training batches to identify which operation produces NaN.
P-06 - SMOTE Synthetic Samples Degrade VAE Reconstruction
Symptom
Training reconstruction loss is lower than expected but validation reconstruction loss is much higher (gap > 0.05).
EXPLOIT and MALWARE class samples have substantially higher reconstruction error than NORMALL.
Root cause: The 72,360 SMOTE-synthesised samples are linear interpolations in feature space. They may lie off the true data manifold, particularly for MALWARE (20,226 synthetics from only 4,774 real seeds with k=5 nearest neighbours). The VAE trained on these synthetic samples may overfit to the interpolated manifold.

Solutions
Option A - Train on real samples only: Use Stage 2 y_train to filter real vs synthetic rows. Stage 2's SMOTE appended synthetic rows after the real training rows. Real rows: indices 0 to 2,295,227. Synthetic rows: 2,295,228 to 2,367,587. Train the VAE on the 2,295,228 real rows only. Then generate z_train for all 2,367,588 rows (real + synthetic) for Stage 5.
Option B - Train on all samples (default): Simpler. The VAE receives more EXPLOIT and MALWARE training examples. The interpolated manifold is still locally valid.
Report which option is used in the paper. FR-08 from Stage 2 specified this as a decision point.
Diagnostic: compute reconstruction loss separately for real vs synthetic samples. If synthetic reconstruction loss is > 2Ś real loss, use Option A.
P-07 - log_var Divergence (Variance Explosion)
Symptom
KL loss grows very large (> 100 per batch) after initial annealing phase.
Model does not converge - loss oscillates or diverges.
Solutions
Clamp log_var to [-10, 10] (see P-05). This is the primary fix.
Reduce learning rate: try lr=5e-4 or lr=1e-4 if oscillation persists.
Increase KL annealing warmup: extend from 10 to 20 epochs to give the encoder more time to stabilise before full KL pressure is applied.
P-08 - class imbalance in Latent Space Even After SMOTE
Symptom
Latent space t-SNE plot shows MALWARE cluster (5,967 real + 20,226 synthetic = 25,000 train) is smaller and less dense than NORMALL cluster (1,813,161 samples).
VAE reconstruction loss for MALWARE is much higher than for NORMALL.
Solutions
Use class-weighted reconstruction loss: multiply the per-sample normalised reconstruction loss by a class weight. Class weights = inverse of class frequency (standard sklearn practice). MALWARE weight  1/0.0106  94Ś. This forces the VAE to prioritise faithful reconstruction of MALWARE samples.
Sample weighting in DataLoader: use a WeightedRandomSampler that oversamples MALWARE and EXPLOIT batches during VAE training, independently of SMOTE.
This is a known limitation: report it in the paper as a threat to validity and note that SMOTE was applied before VAE training to partially mitigate it.

9. Quantum Layer Compatibility - Zero-Problem Guarantee

This section specifies the complete validation protocol that must pass before ANY Stage 3 output is passed to Stage 4. A single angle outside [0, p] or a NaN angle will cause the quantum circuit to produce undefined results that are numerically valid (no error raised) but scientifically meaningless. These failures are silent and catastrophic.

9.1  Why [0, p] Is the Required Range
The VQC uses angle encoding via the RY (Y-axis rotation) gate. The RY gate applies a rotation of angle ? around the Y-axis of the Bloch sphere:

RY(?)|0? = cos(?/2)|0? + sin(?/2)|1?
 
When ? = 0:   state = |0?  (probability of measuring 1 = 0)
When ? = p:   state = |1?  (probability of measuring 1 = 1)
When ? = p/2: state = |+?  (probability of measuring 1 = 0.5 - maximum superposition)
 
Values outside [0, p]:
  ? > p:  The qubit rotates past |1? and begins returning toward |0?.
           RY(p + e) produces a state equivalent to RY(p - e).
           Values in (p, 2p] add no new information - they duplicate [0, p].
  ? < 0:  Equivalent to RY(|?|) with a phase flip. Functionally identical to
           the corresponding positive angle but may produce implementation-specific
           inconsistencies across different quantum simulation backends.
 
Conclusion: [0, p] is the information-maximal, backend-safe range for RY encoding.

    min_dist = float('inf')
    min_pair = None
    for i in range(len(classes)):
        for j in range(i+1, len(classes)):
            ci, cj = int(classes[i]), int(classes[j])
            d = (centroids[ci] - centroids[cj]).norm().item()
            if d < min_dist:
                min_dist = d;  min_pair = (class_names[ci], class_names[cj])
    print(f'  [6/8] Min inter-class centroid dist: {min_dist:.4f}  ({min_pair[0]} vs {min_pair[1]})')
    if min_dist < 0.1:
        print('         WARNING: Very low separation. cVAE or auxiliary loss recommended.')
 
    # 7. No constant dimensions in test set
    test_std = z_test.std(dim=0)
    const_test = (test_std < 1e-4).nonzero(as_tuple=True)[0].tolist()
    if const_test:
        print(f'  [7/8] WARNING: z_test dims {const_test} are near-constant. VQC will not discriminate.')
    else:
        print(f'  [7/8] Test angle diversity:      PASS  (min_std={test_std.min():.4f})')
 
    # 8. Label distribution preserved
    test_labels, test_counts = torch.unique(y_test, return_counts=True)
    expected_classes = {0, 1, 2, 3, 4}
    found_classes = set(test_labels.tolist())
    assert found_classes == expected_classes, \
        f'[8/8] Test set missing classes: {expected_classes - found_classes}'
    print('  [8/8] All 5 classes in test set: PASS')
 
    print(f'=== ALL VALIDATION CHECKS COMPLETE FOR {model_name} ===')
    return {'dim_std': dim_std.tolist(), 'min_class_dist': min_dist,
            'collapsed_dims': collapsed_dims, 'const_test_dims': const_test}

10.1  Saving the Latent Vectors
def extract_and_save_latent(model, data_loader, device, output_path,
                             y_array, col_names=None):
    """
    Extract quantum angle vectors for all samples.
    Writes to Parquet in chunks to avoid memory overflow.
    """
    import pyarrow as pa
    import pyarrow.parquet as pq
 
    col_names = col_names or [f'z_{i}' for i in range(8)]
    schema    = pa.schema([(c, pa.float32()) for c in col_names])
 
    model.eval()
    model.to(device)
    all_angles = []
 
    with torch.no_grad():
        for batch in data_loader:
            x = batch[0].to(device)
            c = None
            if model.cond_dim > 0 and len(batch) > 2:
                y = batch[2].to(device)
                c = torch.zeros(len(y), 5, device=device)
                c.scatter_(1, y.unsqueeze(1), 1.0)
            mu, _ = model.encode(x, c)
            angles = to_quantum_angles(mu).cpu()
            all_angles.append(angles)
 
    z_all = torch.cat(all_angles, dim=0)   # (N, 8)
 
    # Final validation before save
    assert (z_all >= 0).all() and (z_all <= torch.pi).all(), \
        'CRITICAL: Angles outside [0, pi] before save!'
    assert not torch.isnan(z_all).any(), 'CRITICAL: NaN in angles!'
 
    # Write Parquet
    df = pd.DataFrame(z_all.numpy(), columns=col_names)
    df.to_parquet(output_path, index=False)
    print(f'Saved {output_path}: shape={z_all.shape}, '
          f'min={z_all.min():.4f}, max={z_all.max():.4f}')
    return z_all

13. IEEE Publication Standards - How Stage 3 Contributes

Stage 3 contributes three distinct, reportable elements to the paper. This section frames them in the language IEEE reviewers expect.

13.1  Novel Contribution: Sentinel-Masked VAE Loss
The sentinel-masked ELBO with per-sample normalisation by real feature count is a novel adaptation of the standard VAE for multi-source heterogeneous data. Standard VAE loss functions assume all input features are observed. The modification is straightforward but the justification (multi-source NIDS training without compromising any single dataset's representation) is directly applicable to the growing field of federated and multi-source intrusion detection. This is a publishable component of the architecture section.

Suggested paper framing: 'We propose a sentinel-aware ELBO loss that down-weights missing feature positions in the reconstruction term, normalised by the number of observed features per sample, ensuring equitable training contributions from datasets with heterogeneous feature coverage.'

13.2  Ablation Study: VAE-A vs VAE-B
The dual-model experiment (with and without near-zero variance features) produces a quantified ablation result. The improvement in VQC F1-macro from VAE-A to VAE-B - if positive and statistically significant - demonstrates that preprocessing feature quality directly impacts quantum circuit classification performance. This is a specific, novel empirical claim about hybrid quantum-classical systems that no prior NIDS paper has made.

13.3  Quantum-Safe Bottleneck Design
The choice of scaled sigmoid (sigmoid(”) Ś p) over hard clipping or tanh rescaling as the angle bounding function is a design decision with theoretical backing: it preserves gradient flow everywhere in (-8, +8), produces angles that cover (0, p) more uniformly than tanh-rescaled alternatives, and is differentiable at all points. This is worth a paragraph in the paper's methodology section, with a comparison table analogous to the one in Section 3.3 of this document.
